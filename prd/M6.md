# M6 — Files, projects, system, faithful stubs (PRD)

> Status: ⬜ · Depends on: M1 (parallel with M2-M5) · Read `prd/README.md`. Tasks: `PRD.md` (M6-T*).

## Context / why

SmileBASIC programs save/load files and projects, read system variables, edit program
source at runtime (PRG* family), and interact with device features (mic, motion, wireless)
that we must reproduce *faithfully including their limitations*. This milestone also
finalizes multi-slot semantics (EXEC/USE/CALL/COMMON DEF). Storage must work on both
native (filesystem) and wasm (IndexedDB).

## Goal (Definition of Done)

- A storage layer (native FS / wasm IndexedDB) with an extdata-compatible layout.
- File commands (SAVE/LOAD/FILES/DELETE/RENAME/COPY/CHKFILE/PROJECT) per spec.
- System variables behave per spec (incl. faithful FREEMEM/HARDWARE models).
- PRG* source-editing commands work; mic/motion/MP behave with the right
  availability errors; DIALOG/FONTDEF/CLIPBOARD implemented.
- Multi-slot + COMMON DEF + EXEC/USE/CALL semantics complete.

## Reference sources
- Specs: `category` Files / Source-manip / Various input / Advanced control in
  `spec/instructions/`; `spec/reference/sysvars.yaml`, `errors.yaml` (36/37/38/43/44/45/46).
- Docs: `sb-docs/smilebasic-3/manual/` file/project topics.
- osb: `project.d` (Projects, Resource, FileName), `program.d` (Program/Slot, PRG* edits),
  `systemvariable.d`, `dialog.d`, `petitcomputer.d` (slots, EXEC/USE), `compiler.d`
  (MIC/MOTION/MP sysvars stubbed to 0).
- Disassembly: file format details + exact sysvar values where they matter.

## Architecture decisions
- `sb-core` defines a **`Storage` trait** (list/read/write/delete/rename); native and wasm
  implement it. Keep actual I/O out of `sb-core`. The extdata-compatible layout lets the
  oracle harvest interoperate (O-T3).
- "Faithful stub" means: reproduce the *observable behavior* of unavailable features
  (e.g. MIC without `XON MIC` → errnum 36; MP sysvars → fixed values), not a real
  implementation.

## Tasks

### M6-T1 — Storage abstraction
- **Files:** `crates/sb-core/src/storage.rs` (new, trait); native impl in
  `crates/sb-platform-native`, wasm impl in `crates/sb-platform-wasm`.
- **Approach:** trait for project/file CRUD with SB's resource types (TXT/DAT/GRP/etc.) and
  an extdata-compatible on-disk layout. Provide an in-memory impl for tests.
- **Acceptance:** in-memory storage tests; native writes/reads round-trip; wasm uses
  IndexedDB.
- **Fixtures:** `harness/corpus/sbsave/files/<KEY>/{TXT,DAT}/` is already laid out
  extdata-style (mirrors SmileBASIC `PROJECTS/<name>/{TXT,DAT}/`) — drop it in as a ready
  project tree for LOAD/FILES tests. `DAT/*` are real `PCBN0001` blobs (M6-T2 LOAD fuel).
  Regenerate via `python3 tools/extract_sbsave.py`; format spec is in that tool's docstring.

### M6-T2 — File commands
- **Files:** `crates/sb-core/src/builtins/files.rs` (new).
- **Approach:** SAVE/LOAD (TXT/DAT/program), FILES (listing), DELETE, RENAME, COPY,
  CHKFILE, PROJECT (current project). Errors: 35 (illegal format), 46 (load failed). → M6-T1
- **Acceptance:** per-spec tests against in-memory storage; SAVE then LOAD round-trips a
  string/array; FILES lists correctly.

### M6-T3 — System variables
- **Files:** `crates/sb-core/src/sysvars.rs`.
- **Approach:** implement all sysvars per `spec/reference/sysvars.yaml`: DATE$/TIME$
  (deterministic clock injection for tests), MAINCNT (from M4 clock), VERSION
  (&HXXYYZZZZ for 3.6.0), FREEMEM (faithful memory model), RESULT, ERR*/PRGSLOT/CALLIDX,
  TABSTEP/SYSBEEP (writable). → M4-T3
- **Acceptance:** sysvar reads match spec; writable ones (TABSTEP/SYSBEEP) take effect;
  VERSION encodes 3.6.0; DATE$/TIME$ deterministic under an injected clock.

### M6-T4 — Source-edit (PRG*)
- **Files:** `crates/sb-core/src/builtins/prg.rs` (new).
- **Approach:** PRGEDIT (select edit target), PRGGET$/PRGSET/PRGINS/PRGDEL, PRGNAME$,
  PRGSIZE. Enforce errnum 38 (use PRGEDIT first). → M1 program model
- **Acceptance:** PRG* round-trip edits a program slot's source; errnum 38 when misused.

### M6-T5 — Misc + faithful limitation stubs
- **Files:** `builtins/dialog.rs`, `builtins/device.rs` (new); `dialog`/`fontdef`.
- **Approach:** DIALOG (RESULT), FONTDEF, CLIPBOARD; MIC family + MOTION (GYRO/ACCEL)
  with availability gating (XON MIC/MOTION) → errnum 36/37; MP wireless family + sysvars
  as faithful stubs (fixed values, like SB offline). DIRECT-mode/program-only restrictions
  → errnum 43/44/45. → M6-T1
- **Acceptance:** DIALOG sets RESULT; MIC/MOTION without XON → 36/37; MP sysvars match
  offline values; DIRECT-only commands error in program (43) and vice versa (44).

### M6-T6 — Multi-slot semantics
- **Files:** `crates/sb-core/src/vm.rs`.
- **Approach:** finalize 4 slots + shared COMMON DEF; EXEC (run another slot), USE (mark
  slot), CALL (call by name), cross-slot GOTO/GOSUB and variable scoping. → M1-T6
- **Acceptance:** programs spanning slots with COMMON DEF run; EXEC/USE/CALL behave per
  spec; CALLIDX correct in SPFUNC/BGFUNC callbacks.

- FREEMEM exact model — osb returns a constant; confirm whether 3.6.0 varies and what
  programs depend on it.
