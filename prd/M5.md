# M5 — Audio (MML) (PRD)

> Status: ⬜ · Depends on: M1 (parallel with M2-M4) · Read `prd/README.md`. Tasks: `PRD.md` (M5-T*).

## Context / why

Audio is where `osb` is weakest (it's stubbed), so this milestone leans hardest on the
docs + disassembly. SmileBASIC plays music via **MML** (Music Macro Language) across 16
channels with instruments, drums, PSG, user waveforms, and envelopes, plus BEEP/TALK and
sound effects. Faithfulness means matching the synth output closely enough to pass
sample/spectral diffs against the oracle.

## Goal (Definition of Done)

- A complete MML parser per the spec, and a synth that renders BGM to PCM.
- BGM/BEEP/TALK/effect commands implemented per spec.
- Audio backends: cpal (native), WebAudio (wasm).
- **Deterministic gate = MML → note-events** (pitch/duration/volume/envelope) + synth parameter
  tables, authored from docs + disassembly and replayed without the emulator. Per **O-T7**, real
  emulator audio is NOT a deterministic golden (no SB render-to-file; real-time/timing-dependent),
  so oracle audio (`sb_audio.py` → WAV) is a **manual reference / loose spectral check only**, never
  a frozen CI fixture. Our synth renders PCM deterministically off the frame clock for its own tests.

## Reference sources
- Specs: `category: Sound` (18) in `spec/instructions/`; **`spec/reference/mml.yaml`**
  (the full MML grammar — channels :0-15, tempo, lengths, gate Q, pitch/octave, key N,
  volume/pan, envelope @E/@ER, instruments @0-127, drums @128/@129, PSG @144-151, user
  waveforms @224-255, @256 BEEP, modulation/vibrato/tremolo/autopan, repeats, MML vars
  $0-7, macros).
- Docs: `sb-docs/smilebasic-3/manual/sound-instructions.md`, `reference/mml.md`.
- Disassembly: the synth/MML interpreter + exact sample rate, mixing, envelope curves,
  instrument tables. This is the authoritative source since osb has none.

## Architecture decisions
- Render audio deterministically off the **frame clock** (M4-T3) so tests are reproducible:
  given an MML string + N frames, produce identical PCM. The live backend just consumes
  that PCM. Keep synth in `sb-audio` (no device deps); backends in platform crates or
  feature-gated.
- Decide the internal sample rate from the disassembly (placeholder constant exists in
  `sb-audio`); resample for the host as needed.

## Tasks

### M5-T1 — MML parser
- **Files:** `crates/sb-audio/src/mml.rs` (new).
- **Approach:** tokenize + parse MML per `spec/reference/mml.yaml` into a per-channel event
  stream (notes, durations, control changes, envelope/instrument selects, repeats `[ ]n`,
  macros `{Label=…}`, MML vars). Validate → errnum 47 (Illegal MML) on bad input.
- **Acceptance:** parse tests for every MML construct in the reference; malformed MML →
  errnum 47; macro/repeat expansion correct.

### M5-T2 — Synth engine
- **Files:** `crates/sb-audio/src/synth.rs` (new), `instruments.rs` (new).
- **Approach:** render the event stream to PCM: per-channel oscillators/samples, ADSR
  envelopes (@E/@ER), instruments @0-127, drum sets @128/@129, PSG @144-151, user
  waveforms @224-255, modulation/vibrato/tremolo/autopan, volume/pan. Mix 16 channels.
  Reproduce SB's exact timing (tempo T, length L, gate Q). → M5-T1
- **Acceptance:** given seed MML, output PCM is deterministic and matches an oracle golden
  WAV within tolerance (sample or spectral distance documented).

### M5-T3 — BGM commands
- **Files:** `crates/sb-core/src/builtins/sound.rs` (new).
- **Approach:** BGMPLAY (string MML or preset id), BGMSET/BGMSETD (define user BGM),
  BGMSTOP, BGMCHK, BGMVAR, BGMVOL, BGMCLEAR. Manage playback state on the frame clock. → M5-T2
- **Acceptance:** BGMPLAY/BGMCHK/BGMSTOP per-spec tests; user-defined BGM (BGMSET) plays;
  BGMVAR read/write.

### M5-T4 — SFX / voice
- **Files:** `sound.rs`, `crates/sb-audio/src/effects.rs` (new).
- **Approach:** BEEP (preset SFX + pitch/vol/pan), TALK/TALKCHK/TALKSTOP (TTS — confirm
  feasibility; may be a faithful stub if device-specific), effects EFCSET/EFCON/EFCOFF/
  EFCWET, WAVSET/WAVSETA (user waveform definition). → M5-T2
- **Acceptance:** BEEP variants match oracle; effect on/off/wet changes output; WAVSET
  feeds @224-255.

### M5-T5 — Audio backend
- **Files:** `crates/sb-platform-native` (cpal), `crates/sb-platform-wasm` (WebAudio).
- **Approach:** stream the synth's PCM to the host device; handle WebAudio scheduling vs
  rAF/VSYNC so playback stays in sync with the frame clock. → M5-T2
- **Acceptance:** audio plays on native + in-browser; no buffer underruns at 60 fps; sync
  with on-screen frame counter.

### M5-T6 — RE timing + golden harvest
- **Files:** spec updates; `harness/oracle/audio.py`; `harness/corpus/golden/audio/`.
- **Approach:** finalize exact sample rate/mixing/envelope from the disassembly; harvest
  oracle audio (needs O-T7) for the MML test set; wire sample/spectral diff into
  `replay.py`. Raise sound specs to `disassembled`/`hw_verified`. → M5-T5, O-T7
- **Acceptance:** committed golden audio; deterministic diff replay in CI (no emulator).

## Milestone verification
```bash
cargo test -p sb-audio -p sb-core
cargo run -p sb-platform-native --bin sb -- <a BGM demo>.sb3
python3 harness/diff/replay.py     # golden audio diffs pass
```

## Risks / open questions
- TALK (TTS) may be hard to reproduce bit-exactly; decide faithful-stub vs best-effort and
  document.
- WebAudio timing/latency vs the deterministic frame clock — keep render deterministic,
  treat the device as a sink.
- Exact instrument/waveform tables must come from the disassembly; budget RE time.
