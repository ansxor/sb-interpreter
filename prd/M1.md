# M1 ‚Äî Core VM + a real window (PRD)

> Status: ‚¨ú gated on S ¬∑ Depends on: M0 ¬∑ Read `prd/README.md` first. Tasks: `PRD.md` (M1-T*).
> NOTE: `lexer.rs`/`ast.rs` exist from a **pre-spec-first attempt** (osb-port, ASCII-only
> identifiers). Rewrite/validate them spec-first against the disassembly + oracle ‚Äî do not
> trust them. osb is a STRUCTURAL reference only (see prd/README.md fidelity rule).

## Context / why

This is the heart of the project: the SmileBASIC language running for real. By the end
of M1 a program like `PRINT FLOOR(-2.1)` executes and you can **watch it in a window**,
and the result is validated against the oracle. Everything after M1 (graphics, sprites,
audio) hangs off this core. We bring up a window *alongside* the core (user's choice) so
there's visual feedback early, but the core must be bit-exact, not a toy.

## Goal (Definition of Done)

- Source ‚Üí lexer ‚Üí parser ‚Üí compiler ‚Üí bytecode ‚Üí stack VM, all in `sb-core`.
- Core language works: variables (int/double/string + suffix typing), 1‚Äì4D arrays,
  operators with SB precedence, IF/FOR/WHILE/REPEAT/GOTO/GOSUB/ON, DEF/COMMON DEF,
  DATA/READ/RESTORE, INPUT.
- Math + string builtins per their specs; PRINT/LOCATE/COLOR/CLS render to a 50√ó30
  console ‚Üí framebuffer ‚Üí window (native + wasm).
- TinyMT RNG bit-exact (validated against `otya_test.sb3` RND asserts).
- Errors raise the correct `ERRNUM`/`ERRLINE`.
- `cargo test` executes every `spec/tests/` overlay + `corpus/cases` through sb-core and
  runs `otya_test.sb3` via an `ASSERT__` test builtin. Native + wasm builds stay green.

## Scope

**In:** the language core, math/string/console builtins, RNG, console rendering, window,
headless runner, conformance wiring.
**Out (later):** graphics drawing (M2), sprites/BG (M3), real input devices/timing (M4 ‚Äî
M1 may stub BUTTON/VSYNC as no-ops to let programs run), audio (M5), files (M6).

## Architecture decisions

- **Opcode representation:** prefer a flat `enum Op { ... }` + `Vec<Op>` with a `match`
  dispatch loop over osb's object-per-opcode design (better for Rust/wasm + serialization
  + determinism). Keep opcode *semantics* faithful to `VM.d`.
- **Stack machine** with explicit operand stack, frame base pointer, pc, and a slot index
  (SB has 4 program slots + shared COMMON functions). Mirror `VM.d`'s frame layout.
- **Strings** are mutable UTF-16 (`Value::Str`). Arrays are reference types (`Rc<RefCell<‚Ä¶>>`
  or an arena+handle; pick one and document it ‚Äî must be wasm-safe, no threads).
- **No I/O in `sb-core`.** The console writes into an in-memory grid; rendering to pixels
  lives in `sb-render`; the window lives in `sb-platform-*`.

### M1-T9 ‚Äî TinyMT RNG ‚úÖ
> Done. `crates/sb-core/src/rng.rs`: a bit-exact **TinyMT32** (canonical default params
> `mat1=0x8f7011ee mat2=0xfc78ff1f tmat=0x3793fdff`) with the 8 independent series (seed IDs
> 0‚Äì7). `RANDOMIZE seed_id, seed_value` = `tinymt32_init(seed)` with **no** extra discard
> draw (osb's `random.d` adds a `popFront`; real SB does not ‚Äî that would shift the sequence,
> disasm seed path `@0x26f580`). `RND(max)` draws one raw 32-bit word and reduces by plain
> `raw % max` (reduce helper `@0x1fd4e8` is a software unsigned modulo, no rejection redraw ‚Äî
> empirically confirmed). `RNDF()` is the 53-bit double from **two** draws,
> `((draw1>>5)*2^26 + (draw2>>6)) * 2^-53` in `[0,1)` (RNDF core `@0x1eac60`: `lsr #5`/`lsr
> #6`, consts `2^26`=`0x4190‚Ä¶`, `2^-53`=`0x3CA0‚Ä¶`). The three builtins live in the VM
> (`Vm::call_rng`) since they mutate series state, with arg validation per spec (count‚Üí4,
> string‚Üí8, seed/max range‚Üí10); registered in `BUILTIN_NAMES` so the compiler treats them as
> calls. **Verified bit-for-bit against the `otya_test.sb3` golden AND re-harvested via
> sb-oracle 2026-06-23:** after `RANDOMIZE 0,1`, `RND(100)` ‚Üí 89,33,33,52, then `RNDF(0)` =
> 0.836095, then `RND(100)` = 66; `RANDOMIZE 5,1`‚Üí`RND(5,100)` = 89. Folded those into
> rnd/rndf/randomize.yaml as `hw_verified` tests (confidence bumped disassembled‚Üíhw_verified;
> the seeded sequence is now a frozen deterministic golden ‚Äî the un-seeded entropy path stays
> non-deterministic, fallback-seeded in `sb-core` for wasm/determinism). 5 engine unit tests +
> 4 VM-program integration tests (full compile‚ÜíVM‚Üírng path). The `RND(0)`/`max==0` degenerate
> (modulo-by-zero guard ‚Üí 0) is the only oracle-queued edge. ‚Üí M1-T7, S-T1
