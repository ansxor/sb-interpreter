# M1 — Core VM + a real window (PRD)

> Status: ✅ done (T1–T14) · Depends on: M0 · Read `prd/README.md` first. Tasks: `PRD.md` (M1-T*).
> NOTE: `lexer.rs`/`ast.rs` exist from a **pre-spec-first attempt** (osb-port, ASCII-only
> identifiers). Rewrite/validate them spec-first against the disassembly + oracle — do not
> trust them. osb is a STRUCTURAL reference only (see prd/README.md fidelity rule).

## Context / why

This is the heart of the project: the SmileBASIC language running for real. By the end
of M1 a program like `PRINT FLOOR(-2.1)` executes and you can **watch it in a window**,
and the result is validated against the oracle. Everything after M1 (graphics, sprites,
audio) hangs off this core. We bring up a window *alongside* the core (user's choice) so
there's visual feedback early, but the core must be bit-exact, not a toy.

## Goal (Definition of Done)

- Source → lexer → parser → compiler → bytecode → stack VM, all in `sb-core`.
- Core language works: variables (int/double/string + suffix typing), 1–4D arrays,
  operators with SB precedence, IF/FOR/WHILE/REPEAT/GOTO/GOSUB/ON, DEF/COMMON DEF,
  DATA/READ/RESTORE, INPUT.
- Math + string builtins per their specs; PRINT/LOCATE/COLOR/CLS render to a 50×30
  console → framebuffer → window (native + wasm).
- TinyMT RNG bit-exact (validated against `otya_test.sb3` RND asserts).
- Errors raise the correct `ERRNUM`/`ERRLINE`.
- `cargo test` executes every `spec/tests/` overlay + `corpus/cases` through sb-core and
  runs `otya_test.sb3` via an `ASSERT__` test builtin. Native + wasm builds stay green.

## Scope

**In:** the language core, math/string/console builtins, RNG, console rendering, window,
headless runner, conformance wiring.
**Out (later):** graphics drawing (M2), sprites/BG (M3), real input devices/timing (M4 —
M1 may stub BUTTON/VSYNC as no-ops to let programs run), audio (M5), files (M6).

## Architecture decisions

- **Opcode representation:** prefer a flat `enum Op { ... }` + `Vec<Op>` with a `match`
  dispatch loop over osb's object-per-opcode design (better for Rust/wasm + serialization
  + determinism). Keep opcode *semantics* faithful to `VM.d`.
- **Stack machine** with explicit operand stack, frame base pointer, pc, and a slot index
  (SB has 4 program slots + shared COMMON functions). Mirror `VM.d`'s frame layout.
- **Strings** are mutable UTF-16 (`Value::Str`). Arrays are reference types (`Rc<RefCell<…>>`
  or an arena+handle; pick one and document it — must be wasm-safe, no threads).
- **No I/O in `sb-core`.** The console writes into an in-memory grid; rendering to pixels
  lives in `sb-render`; the window lives in `sb-platform-*`.

### M1-T3 — Parser ✅
> Done. `crates/sb-core/src/parser.rs`: recursive-descent statement parser + precedence-
> climbing expression parser + parse-time constant folding, over the M1-T1 token stream into
> the M1-T2 AST. Two SmileBASIC facts shaped it and differ from osb (whose `parser.d` is the
> structural reference): the lexer does **not** pre-classify keywords (control words *and*
> commands arrive as `Ident`, so dispatch is by upper-cased name here, and a suffixed name

> assignment at statement start via `parse_operand` rank ≤ 2, equality `BinOp::Eq` inside any
> expression). Const folding uses runtime numeric semantics (i32 wrap for integer `+ - *`,
> f64 for any-real / `/`, integer `DIV`/`MOD`/`AND`/`OR`/`XOR`; never folds a divide-by-zero,
> shifts, comparisons, or `&&`/`||` — those go to the VM where verified). Covers the full AST
> statement set (assign/array-assign/ref-assign, paren- and space-form command calls with
> `OUT`, PRINT separators, IF single/multi-line + ELSEIF/ELSE + `IF c GOTO`/`THEN @L`,
> FOR/WHILE/REPEAT, GOTO/GOSUB/RETURN/ON, DIM/VAR, DEF/COMMON DEF, DATA/READ/RESTORE,
> INPUT/LINPUT, INC/DEC→negated-delta, SWAP, OPTION, USE/EXEC, labels). Malformed input →
> `ParseError { errnum: 3 }` with a `SourceLoc`. 26 parser unit tests (precedence, folding,
> unary, postfix/index/call/ref, every statement form, errnum-3 cases). Hardened against a
> 3,019-program `sbsave` corpus parse-sweep (~78%; remainder = non-program text files, SB4
> BIG programs, and lexer-level gaps queued in `HARVEST_QUEUE.md`: `#const` in DATA, `0#`/`13e4`
> literals, single-line-IF-vs-NEXT extent, stray/unbalanced block keywords). `^` power op
> omitted (no caret token; rank queued). → M1-T2, S-T6

- **Files:** `crates/sb-core/src/parser.rs`.

### M1-T9 — TinyMT RNG ✅
> Done. `crates/sb-core/src/rng.rs`: a bit-exact **TinyMT32** (canonical default params
> `mat1=0x8f7011ee mat2=0xfc78ff1f tmat=0x3793fdff`) with the 8 independent series (seed IDs
> 0–7). `RANDOMIZE seed_id, seed_value` = `tinymt32_init(seed)` with **no** extra discard
> draw (osb's `random.d` adds a `popFront`; real SB does not — that would shift the sequence,
> disasm seed path `@0x26f580`). `RND(max)` draws one raw 32-bit word and reduces by plain
> `raw % max` (reduce helper `@0x1fd4e8` is a software unsigned modulo, no rejection redraw —
> empirically confirmed). `RNDF()` is the 53-bit double from **two** draws,
> `((draw1>>5)*2^26 + (draw2>>6)) * 2^-53` in `[0,1)` (RNDF core `@0x1eac60`: `lsr #5`/`lsr
> #6`, consts `2^26`=`0x4190…`, `2^-53`=`0x3CA0…`). The three builtins live in the VM
> (`Vm::call_rng`) since they mutate series state, with arg validation per spec (count→4,
> string→8, seed/max range→10); registered in `BUILTIN_NAMES` so the compiler treats them as
> calls. **Verified bit-for-bit against the `otya_test.sb3` golden AND re-harvested via
> sb-oracle 2026-06-23:** after `RANDOMIZE 0,1`, `RND(100)` → 89,33,33,52, then `RNDF(0)` =
> 0.836095, then `RND(100)` = 66; `RANDOMIZE 5,1`→`RND(5,100)` = 89. Folded those into
> rnd/rndf/randomize.yaml as `hw_verified` tests (confidence bumped disassembled→hw_verified;
> the seeded sequence is now a frozen deterministic golden — the un-seeded entropy path stays
> non-deterministic, fallback-seeded in `sb-core` for wasm/determinism). 5 engine unit tests +
> 4 VM-program integration tests (full compile→VM→rng path). The `RND(0)`/`max==0` degenerate
> (modulo-by-zero guard → 0) is the only oracle-queued edge. → M1-T7, S-T1
- **Files:** `crates/sb-core/src/rng.rs` (new, port `tinymt32.d`), RND/RNDF/RANDOMIZE in
  `builtins/math.rs`.
- **Approach:** 8 independent TinyMT engines (SB's seed series 0-7). Reproduce the RNDF
  double-draw quirk (`random.d:54`). RANDOMIZE seeding must match. → M1-T7
- **Acceptance:** the RND/RNDF asserts in `otya_test.sb3` pass exactly (e.g. after

    then converts the consts to values). **Form 1** (`COPY dest [,dest_offset], src [[,src_offset],
    count]`) copies array→array; the optional offsets/form are disambiguated by argument TYPE (a
    numeric in slot 2 = `dest_offset`; the source is then an array (form 1) or a string `"@Label"`
    (form 2)). A too-small **1D** destination auto-extends; values coerce to the destination's
    element type (so `COPY`/`FILL` of a numeric into a string array — or vice versa — raises Type
    mismatch 8, coerced before any write so the destination is left untouched on mismatch).
    `FILL array,value[,offset[,count]]` overwrites an element range with a value and never changes
    the length (offset/count past the end → Subscript out of range 31). The arrays reach these
    builtins as shared `Rc` handles via the normal `PushVar` path (a bare array name), so the
    mutations are visible to the caller — no special operand compilation needed (unlike the
    stack ops). Errnums: wrong arg count → 4; non-array operand / element-type mismatch → 8;
    out-of-range offset/count (form 1) → 10; COPY undefined label → 14; COPY short DATA → 13;
