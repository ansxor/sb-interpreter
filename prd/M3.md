# M3 — Sprites & BG (PRD)

> Status: ⬜ · Depends on: M2 · Read `prd/README.md` first. Tasks: `PRD.md` (M3-T*).

## Context / why

Sprites and background (BG) tilemaps are how SmileBASIC games draw moving and tiled
graphics. Both are rich subsystems (attributes, animation, collision, linking, variables).
They plug into the M2 compositor's sprite/BG layer slots. `osb`'s `sprite.d`/`bg.d` are a
strong behavioral template but target 3.5.0 — confirm 3.6.0 specifics against the docs and
disassembly.

## Goal (Definition of Done)

- Sprite system: create/manage 0-511 sprites, attributes (display/rotation/flip/additive),
  position/char/scale/rotation/color/home/page, animation, collision, linking, SPVAR.
- BG system: multiple BG layers/screens, tile put/get/fill, scroll/rotate/scale/color,
  animation, collision, BG functions.
- Both composite correctly into the framebuffer; golden PNG diffs vs oracle pass.

## Reference sources
- Specs: `category: Sprites` (27) and `category: BG` (24) in `spec/instructions/`;
  `spec/reference/constants.yaml` (#SPSHOW/#SPROT*/#SPREV*/#SPADD, #BGROT*/#BGREV*,
  collision #CHK*).
- Docs: `sb-docs/smilebasic-3/manual/{sprites,bg-backgrounds}.md`.
- osb: `sprite.d` (SpriteData, SpriteAttr, SpriteAnimState/Data, SpriteCollision/HitInfo,
  SPLINK, SPVAR; default defs `spdef.csv`), `bg.d` (BGChip layers).
- Disassembly: confirm attribute bitfields, collision math, animation timing, BG transform
  order vs the oracle.

- **Done (2026-06-23):** `SpriteAnim`/keyframe model + `SpriteState::{set_anim,tick,link,

  SPFUNC orchestrated by the VM for DATA/array/`@label` access; the rest pure over the table).
  The frame clock (VSYNC/WAIT) drives `tick`, so animations advance deterministically; the V

### M3-T3 — Sprite collision
- **Files:** `sprite.rs`.
- **Approach:** SPCOL (enable + set collision mask/scale), SPCHK, SPHITSP (sprite-sprite),
  SPHITRC (rect), SPHITINFO, SPCOLVEC; SPDEF (definition templates, seed from `spdef.csv`).
  Collision flags #CHKXY/#CHKZ/#CHKUV/#CHKI/#CHKR/#CHKS/#CHKC/#CHKV. → M3-T1
- **Acceptance:** collision tests (overlapping vs not, with scale/rotation, with mask
  flags) match oracle; deterministic.
- **Done (2026-06-23):** Collision model added to `crates/sb-render/src/sprite.rs`
  (per-slot `col_*` detection rect/scale-adjust/mask/velocity + a shared `HitInfo` record +
  a `SpdefEntry` template table) and the builtins in `crates/sb-core/src/builtins/sprite.rs`
  wired through the VM: **SPCOL** (7 forms — setters 1-3 incl. the `,,`-skip mask; OUT
  getters 4-7), **SPCOLVEC** (auto/explicit vector), **SPCHK** (animation-status bitmask),
  **SPHITSP** (vs-all / vs-one boolean / vs-range), **SPHITRC** (rect vs-all/one/range, the
  4/5/6 + 7/8/9 mask+movement arg shapes), **SPHITINFO** (1/3/5/9-var OUT read of the hit
  record), and **SPDEF** (reset / define / array+`@label` bulk / OUT read / copy-with-adjust;
  `SPSET` form 1 now copies the template's rect+home+attr). **SPOFS** landed as the
  positioning glue collision needs (set 3/4-arg with `,,`-skip + OUT get). Collision is
  **static AABB** (detection rect at the SPOFS/SPLINK position, strict-inequality overlap,
  32-bit AND-mask filter); the swept/time math is `time=0` (= position-at-detection). Tests:
  the specs' hw_verified hit/miss + error cases (folded into `IN_SCOPE_SPRITES`) +
  `harness/corpus/cases/sprite_collision.yaml` e2e (overlap/separation/mask/range/explicit-
  rect, SPCHK running/stopped, SPHITINFO positions, SPCOL OUT round-trip, SPDEF
  define/read/reset/copy/array + `SPSET`-uses-template-rect) + 12 new sb-render unit tests,
  all green; 3,329-program corpus sweep still 0 panics. **Oracle-pending** (HARVEST_QUEUE
  S-T8d): swept-collision time/coords for moving objects, scaled-collision exactness, the
  exact SPDEF form-2-vs-6 disambiguation, SPDEF non-default read-back, and the real
  `spdef.csv` default-template rects (seeded as 16×16 here). Remaining transform setters
  (SPROT/SPSCALE/SPHOME/SPCHR/SPPAGE/SPCOLOR) and the visible-render golden block on a later
  increment + the framebuffer oracle (O-T6 / M3-T6).
