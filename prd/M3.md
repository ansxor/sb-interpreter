# M3 — Sprites & BG (PRD)

> Status: ⬜ · Depends on: M2 · Read `prd/README.md` first. Tasks: `PRD.md` (M3-T*).

## Context / why

Sprites and background (BG) tilemaps are how SmileBASIC games draw moving and tiled
graphics. Both are rich subsystems (attributes, animation, collision, linking, variables).
They plug into the M2 compositor's sprite/BG layer slots. `osb`'s `sprite.d`/`bg.d` are a
strong behavioral template but target 3.5.0 — confirm 3.6.0 specifics against the docs and
disassembly.

## Goal (Definition of Done)

- Sprite system: create/manage 0-511 sprites, attributes (display/rotation/flip/additive),
  position/char/scale/rotation/color/home/page, animation, collision, linking, SPVAR.
- BG system: multiple BG layers/screens, tile put/get/fill, scroll/rotate/scale/color,
  animation, collision, BG functions.
- Both composite correctly into the framebuffer; golden PNG diffs vs oracle pass.

## Reference sources
- Specs: `category: Sprites` (27) and `category: BG` (24) in `spec/instructions/`;
  `spec/reference/constants.yaml` (#SPSHOW/#SPROT*/#SPREV*/#SPADD, #BGROT*/#BGREV*,
  collision #CHK*).
- Docs: `sb-docs/smilebasic-3/manual/{sprites,bg-backgrounds}.md`.
- osb: `sprite.d` (SpriteData, SpriteAttr, SpriteAnimState/Data, SpriteCollision/HitInfo,
  SPLINK, SPVAR; default defs `spdef.csv`), `bg.d` (BGChip layers).
- Disassembly: confirm attribute bitfields, collision math, animation timing, BG transform
  order vs the oracle.

## Tasks

### M3-T1 — Sprite core
- **Files:** `crates/sb-render/src/sprite.rs` (new); `crates/sb-core/src/builtins/sprite.rs` (new).
- **Approach:** sprite table (0-511), each with def (U,V,W,H), pos (SPOFS, incl. Z), char
  (SPCHR), scale (SPSCALE), rotation (SPROT), color (SPCOLOR), home (SPHOME), page
  (SPPAGE), attribute bitfield (SPSET attr: b00 display, b01-02 rot, b03 H-flip, b04
  V-flip, b05 additive). SPSET (all 6 forms incl. OUT IX allocation + range), SPCLR,
  SPSHOW/SPHIDE, SPUSED. → M2-T4
- **Acceptance:** SPSET forms' tests pass (incl. OUT allocation, -1 when full); a sprite
  renders at the right place/orientation (golden diff).

### M3-T2 — Sprite animation / link / vars
- **Files:** `sprite.rs`.
- **Approach:** SPANIM (multi-target animation data), SPSTART/SPSTOP, SPFUNC + CALLIDX
  callback model, SPVAR (per-sprite variables), SPLINK/SPUNLINK (parent-child transforms).
  Match animation frame timing to SB (per-frame advance). → M3-T1
- **Acceptance:** animation advances correctly over frames (deterministic with a fixed
  frame clock); linked sprites inherit transforms; SPVAR read/write; errnum 39/40 for bad
  animation data.

### M3-T3 — Sprite collision
- **Files:** `sprite.rs`.
- **Approach:** SPCOL (enable + set collision mask/scale), SPCHK, SPHITSP (sprite-sprite),
  SPHITRC (rect), SPHITINFO, SPCOLVEC; SPDEF (definition templates, seed from `spdef.csv`).
  Collision flags #CHKXY/#CHKZ/#CHKUV/#CHKI/#CHKR/#CHKS/#CHKC/#CHKV. → M3-T1
- **Acceptance:** collision tests (overlapping vs not, with scale/rotation, with mask
  flags) match oracle; deterministic.

### M3-T4 — BG core
- **Files:** `crates/sb-render/src/bg.rs` (new); `crates/sb-core/src/builtins/bg.rs` (new).
- **Approach:** BG layers/screens (BGSCREEN, BGPAGE), tile ops BGPUT/BGGET/BGFILL/BGCLR,
  transforms BGOFS (scroll, incl. Z)/BGROT/BGSCALE, BGCOLOR, BGSHOW/BGHIDE/BGHOME, BGCLIP.
  Tile attribute bitfields (#BGROT*/#BGREV*). → M2-T4
- **Acceptance:** BG tile placement + scroll/rotate/scale render correctly (golden diffs);
  per-spec tests pass.

### M3-T5 — BG animation / extras
- **Files:** `bg.rs`.
- **Approach:** BGANIM, BGFUNC + CALLIDX, BGSTART/BGSTOP, BGVAR, BGCHK, BGCOORD
  (screen↔tile coords), BGCOPY, BGLOAD/BGSAVE. → M3-T4
- **Acceptance:** animation/coordinate-conversion tests match oracle; load/save round-trip.

### M3-T6 — Composite into framebuffer
- **Files:** `crates/sb-render/src/compositor.rs`.
- **Approach:** wire sprite + BG layers into the M2 compositor with correct Z interleaving
  vs GRP/console; finalize draw order. → M3-T1, M3-T4
- **Acceptance:** full-scene golden PNG (BG + sprites + GRP + console) matches oracle;
  Z-order across all layer types correct.

## Risks / open questions
- Collision exactness (rotated/scaled sprites, mask flags) is subtle — pin with oracle.
- BG transform order (scroll vs rotate vs scale origin) must match SB; confirm via golden
  diffs + disassembly.
