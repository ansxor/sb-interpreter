# sb-oracle — drive real SmileBASIC 3.6.0 in Azahar

Capture ground truth from real SB 3.6.0 to fill `hw_verified` specs (see `prd/oracle.md`).

## What works (verified)

The **typed-command path is reliable** and is the recommended method. Verified end-to-end
against real SB 3.6.0: `FLOOR(3.7)→3`, `FLOOR(-2.1)→-3`, `FLOOR(5)→5`, `7 DIV 2→3`,
`7 MOD 3→1`, `LEN("ABCDE")→5`, `ABS(-9)→9` — each typed into DIRECT mode, executed, and
the result read from an extdata file on disk (no OCR).

```bash
cd .claude/skills/sb-oracle/tools
python3 run_case.py expr 'FLOOR(-2.1)'          # -> -3   (numeric: wraps in STR$)
python3 run_case.py expr 'MID$("ABCDE",2,3)' str  # -> BCD (string: no STR$)
python3 run_case.py cmd 'SAVE"TXT:O",STR$(1+2)'   # run a verbatim command that SAVEs to O
```

## How it works

- **Raise window:** `open -a Azahar` (osascript `activate` is unreliable). Window pinned to
  pos(60,80) size(400,539) so coordinates are stable.
- **Touch:** `cliclick` at screen point `(60+wx, 80+wy)` for window-relative `(wx,wy)`.
  The keymap (`keymap.json`, generated by `gen_keymap.py` from `inputs.json` via
  `window = input + (40,272)`, scale 1.0) is verified accurate.
- **Type a command** (DIRECT mode), **ENTER** to execute. Clear a line with **SHIFT+BACKSPACE**.
- **Capture:** the command `SAVE"TXT:O",<value>` writes a file; SB pops a **"Write file"
  (or overwrite) confirm dialog** → tap **YES** (poll-tap, since render timing varies) →
  read the file off disk (`body[80:-20]`). Result `O` is on-disk `TO` (TXT type prefix `T`).

Tools: `sb_window.py` (raise/tap/type/clear/enter/press/shot), `sb_extdata.py`
(read/`read_result`/write), `run_case.py` (orchestration), `gen_keymap.py` + `inputs.json`.

## extdata file format (cracked)

Path: `~/Library/Application Support/Azahar/sdmc/Nintendo 3DS/<0*32>/<0*32>/extdata/00000000/000016DE/user/###/<NAME>`.
Layout: **80-byte header + UTF-8 source + 20-byte footer**. `header[8:12]` = source length;
`body[80:-20]` = source (footer is HMAC-SHA1 but **reading ignores it**). On-disk name =
type-prefix + in-SB name: TXT→`T` (so `TXT:O`→`TO`).

## Pitfalls
- `LOAD"TXT:name"` needs an output variable (→ "Illegal function call"); use `LOAD"PRG0:name"` for programs.
- Can't chain `LOAD ... :RUN` on one line (→ "Syntax error"); issue RUN separately.
- Don't `clear_line()` between LOAD and RUN (corrupts the next type).
- The terminal can occlude Azahar; `open -a Azahar` right before capture/click.
- Don't hammer the RPC — it destabilizes SB.
