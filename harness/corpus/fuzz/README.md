# Fuzz corpus (M7-T1)

Promoted findings + breadth seeds from the differential fuzzing campaign. These are test
**inputs**, not goldens: the only assertion is that `sb-core` processes them **without
panicking** (a host crash SmileBASIC itself never does). Replayed deterministically in CI by
`crates/sb-core/tests/fuzz_corpus.rs` — no emulator, no network.

Generated by the seeded, spec-signature-driven `harness/fuzz/generator.py`. To reproduce a
file: `python3 harness/fuzz/generator.py <seed> --profile <safe|broad>`. The committed files
are frozen snapshots — if the generator evolves they need not match `generate(seed)` again;
their value is as fixed regression inputs.

## Layout

- `regressions/` — hand-minimized repros of the panics the campaign found. Each crashed
  pre-fix and passes post-fix; **run through the VM** in CI:
  - `gtri_coord_overflow.sb3` — `GTRI` with far-apart i32 vertices overflowed the i32 edge-
    function products (`raster.rs`). Fixed: edge/area math in i128 (a far-corner
    difference reaches ~2^32, so even i64 overflows).
  - `gcopy_span_overflow.sb3` — `GCOPY` with far-apart corners overflowed `(x1-x2).abs()` and
    the `w*h` capacity (`bitmap.rs`). Fixed: i64 span + page-intersection clamp.
  - `val_multibyte_prefix.sb3` — `VAL` of a string whose first char is a full-width glyph
    byte-sliced mid-codepoint in the `&H`/`&B` prefix check (`string.rs`). Fixed: `get(..n)`.
- `safe/` — runtime-safe, guaranteed-terminating programs (safe generator profile: math /
  string / bit functions + operators + bounded `FOR` only). **Run through the VM** in CI.
  `00452.sb3` is the seed that surfaced the `VAL` char-boundary panic.
- `broad/` — the broad profile additionally emits arbitrary scalar-arg command statements,
  which may not terminate (e.g. `VSYNC <huge>`). **Parsed + compiled only** in CI, never run.
  `00013/00104/00179.sb3` are the seeds that surfaced the `GTRI`/`GCOPY` overflows.

## Running the campaign (offline / maintainer)

```bash
cargo build --bin sb-run
python3 harness/diff/run.py --profile safe  --seeds 1000
python3 harness/diff/run.py --profile broad --seeds 1000
```

A non-zero exit means a crash was found; the runner prints the seed + the promote command.
The three-way differential vs the real-hardware oracle (sb-core vs osb vs SmileBASIC 3.6.0 in
Azahar) is the offline maintainer step — like O-T8's live harvest it needs the emulator and is
not part of the hermetic loop.
